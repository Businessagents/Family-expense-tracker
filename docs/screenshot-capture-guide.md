## Setup Instructions

### Device/Emulator Recommendations

**Recommended Resolution:** 1080 x 2340 pixels (9:19.5 aspect ratio)
- This is the standard for modern Android devices
- Google Play Store accepts 1080x1920 to 1080x2400

**Best Options:**
1. **Android Emulator (Recommended)**
   - Device: Pixel 6 or Pixel 7
   - Resolution: 1080 x 2340
   - API Level: 33 or 34 (Android 13/14)

2. **Physical Device**
   - Any modern Android device with 1080p+ display
   - Ensure device is clean (no personal notifications)

## Technical Capture Methods

### Method 1: Android Studio Emulator (Recommended)

**Pros:** Clean screenshots, exact resolution control, no personal data

**Steps:**
1. Open Android Studio
2. Run your app on emulator (Pixel 6, API 33)
3. Navigate to desired screen
4. Click camera icon in emulator toolbar, OR
5. Use keyboard shortcut: `Ctrl+S` (Windows) or `Cmd+S` (Mac)
6. Screenshots saved to: `~/Pictures/Screenshots/` or project `screenshots/` folder

**Settings:**
- In emulator settings, ensure "Take screenshot" saves at full resolution
- Disable emulator frame if you want frameless screenshots

### Method 2: ADB Command Line

**Pros:** Works on both emulator and physical device, automated

**Steps:**
```bash
# Connect device/emulator
adb devices

# Take screenshot
adb shell screencap -p /sdcard/screenshot.png

# Pull to computer
adb pull /sdcard/screenshot.png ~/Desktop/screenshot.png

# Clean up
adb shell rm /sdcard/screenshot.png
```

**Automation Script:**
```bash
#!/bin/bash
# save as capture_screenshots.sh

COUNTER=1
NAMES=(
  "home_dashboard"
  "add_expense"
  "analytics_overview"
  "analytics_category"
  "analytics_trends"
  "balances_settlements"
  "expenses_list"
  "groups_management"
)

for NAME in "${NAMES[@]}"; do
  echo "Capturing ${NAME}..."
  echo "Navigate to the screen and press Enter"
  read

  adb shell screencap -p /sdcard/temp.png
  adb pull /sdcard/temp.png ~/Desktop/${COUNTER}_${NAME}.png
  adb shell rm /sdcard/temp.png

  ((COUNTER++))
  echo "Saved ${NAME}!"
done

echo "All screenshots captured!"
```

### Method 3: Physical Device

**Steps:**
1. Open app and navigate to desired screen
2. Press Power + Volume Down simultaneously
3. Screenshot saved to Gallery/Photos
4. Transfer to computer via:
   - USB cable (File Transfer mode)
   - Google Drive/Dropbox upload
   - Email to yourself

**Important:**
- Ensure device resolution is at least 1080p
- Remove any personal data before capturing
- Crop status bar if needed

## Post-Processing & Enhancement

### Image Editing Tools

**Recommended Tools:**
1. **Figma** (Free) - Best for adding device frames and captions
2. **Canva** (Free tier available) - Templates for app screenshots
3. **Photoshop/GIMP** - Professional editing
4. **Screenshot.rocks** - Online tool for adding device frames

### Enhancement Steps

#### 1. Crop & Resize (if needed)
- Target: 1080 x 2340 px (or your captured resolution)
- Remove status bar if it contains time/battery (optional)
- Keep consistent across all screenshots

#### 2. Add Device Frame (Optional but Recommended)

**Using Figma:**
1. Import screenshot
2. Add device mockup (search "Android phone mockup")
3. Place screenshot inside mockup
4. Export as PNG

**Pre-made Mockups:**
- mockuphone.com
- smartmockups.com
- shotsnapp.com

#### 3. Add Text Overlay (Optional)

**Best Practices:**
- Use large, readable font (min 48px)
- Place text at top or bottom
- Use high contrast (white text on dark overlay)
- Keep it short (1-2 lines max)
- Highlight key feature of that screen

**Example Overlays:**
```
Screenshot 1: "Track Every Expense Effortlessly"
Screenshot 2: "Add Expenses in Seconds"
Screenshot 3: "Understand Your Spending"
Screenshot 4: "Visualize Where Money Goes"
Screenshot 5: "Monitor Trends Over Time"
Screenshot 6: "Settle Up with Ease"
Screenshot 7: "Find Any Expense Instantly"
Screenshot 8: "Collaborate with Groups"
```

#### 4. Optimize File Size

**Requirements:**
- Max file size: 8MB per screenshot (Play Store limit)
- Format: PNG or JPEG
- PNG recommended for crisp text and UI

**Optimization:**
```bash
# Using ImageMagick
convert input.png -quality 95 output.png

# Using pngquant (lossy compression)
pngquant --quality=85-95 input.png -o output.png
```

## Quality Checklist

Before submitting screenshots, verify each one meets these criteria:

### Technical Requirements
- [ ] Resolution: 1080 x 2340 px (or within 1080x1920 to 1080x2400 range)
- [ ] Format: PNG (preferred) or JPEG
- [ ] File size: Under 8MB each
- [ ] Aspect ratio: 9:16 to 9:19.5
- [ ] Color depth: 24-bit or higher
- [ ] No transparency in main content area

### Content Quality
- [ ] No personal or sensitive information visible
- [ ] No real names (unless using test names)
- [ ] No real phone numbers or emails
- [ ] No notification bar with personal info
- [ ] Realistic, professional-looking test data
- [ ] All text is readable and clear
- [ ] No typos in test data
- [ ] Good contrast and lighting

### UI/UX Quality
- [ ] App is in finished state (no loading states)
- [ ] No error messages or broken UI
- [ ] All important features are visible
- [ ] Navigation bar is visible
- [ ] Correct theme applied (dark mode consistent)
- [ ] Icons and images loaded properly
- [ ] No placeholder or "lorem ipsum" text
- [ ] Currency symbols displayed correctly

### Composition
- [ ] Important content centered and visible
- [ ] Not too much empty space
- [ ] Key features highlighted
- [ ] Natural user flow demonstrated
- [ ] Consistent styling across all screenshots
- [ ] Device frame aligned (if using frames)
- [ ] Text overlay readable (if using text)

### Google Play Specific
- [ ] Minimum 2 screenshots (max 8 recommended)
- [ ] Screenshots represent actual app functionality
- [ ] No misleading or exaggerated claims in overlays
- [ ] Complies with Google Play Developer Policy
- [ ] Shows app in actual use cases

## Google Play Store Upload Checklist

When you're ready to upload to Google Play Console:

1. **Navigate to:** App content > Store listing > Screenshots
2. **Upload requirements:**
   - Phone: Upload 2-8 screenshots
   - 7-inch tablet: Optional (can reuse phone screenshots)
   - 10-inch tablet: Optional
3. **Upload order matters:**
   - First 2 screenshots are most important
   - They appear in search results
   - Order from most impressive to least
4. **Recommended upload order:**
   1. Home Dashboard (shows main value)
   2. Analytics Category (visual appeal)
   3. Add Expense (core functionality)
   4. Balances (unique feature)
   5. Groups Management (collaboration feature)
   6. Trends (data insights)
   7. Expenses List (organization)
   8. Profile Security (trust/security)

## Tips for Success

### Do's
- Use realistic, relatable data
- Show the app in actual use
- Maintain consistent currency (USD recommended for global appeal)
- Highlight unique features (balances, groups, analytics)
- Keep screenshots fresh and updated with app changes
- Test on multiple devices before finalizing

### Don'ts
- Don't use "Coming Soon" or incomplete features
- Don't show error states or bugs
- Don't use overly generic stock photos
- Don't make text overlays too busy
- Don't use low-resolution images
- Don't show outdated UI

### Pro Tips
1. **First screenshot is crucial** - Make it your best one showing clear value
2. **Tell a story** - Arrange screenshots to show user journey
3. **A/B test** - Try different arrangements and see what converts better
4. **Localize** - Consider creating screenshots for major languages
5. **Update seasonally** - Refresh data to show current month/year
6. **Show diversity** - Use varied names and scenarios in test data
7. **Competitive analysis** - Check top apps in Finance category for inspiration

## Troubleshooting

### Common Issues

**Issue:** Screenshots too dark on Play Store
- **Solution:** Increase brightness/contrast by 10-15% before uploading

**Issue:** Text too small to read in thumbnails
- **Solution:** Add text overlays with larger font size highlighting key features

**Issue:** Screenshots rejected by Google
- **Solution:** Ensure no copyrighted content, misleading claims, or policy violations

**Issue:** Colors look different on Play Store
- **Solution:** Save in sRGB color space, avoid exotic color profiles

**Issue:** File size too large
- **Solution:** Use PNG compression or convert to JPEG at 90% quality
